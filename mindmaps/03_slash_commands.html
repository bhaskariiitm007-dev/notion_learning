Project-Scoped vs User-Scoped — When to Use Which
Choose the right scope for the right situation
📁 PROJECTUse Project-Scoped when...
Command depends on this project's architecture or DB schema
Whole team should access the same workflow automatically
Commands should be version-controlled alongside codebase
New devs get all workflow commands via git clone
👤 USERUse User-Scoped when...
Command applies generically to any project you work on
Personal preferences: code style, commit message format
You don't want to expose commands to teammates
Reusable across multiple projects without duplication

Core Insight — Stop Repeating Yourself
Every task you repeat more than twice is a candidate for a slash command. create-spec → plan → validate against acceptance criteria is the golden loop.
