Two Types of Skills
Where you store them determines who can use them
👤
Personal Skills
Stored in your home dir. Available across ALL projects. Your private toolkit.
👥
Project Skills
Stored in .claude dir. Scoped to project. Version-controlled & team-shareable.

Common Pitfalls & Pro Tips
Avoid these when building with skills
!
Over-broad skill.md — too many responsibilities in one skill defeats the purpose.
!
Skipping iteration — first draft is rarely perfect. Test, then refine.
!
Wrong directory — team guidelines must live in .claude, not personal home dir.
✓
Name skills clearly — use frontend-design for instant recognition.
✓
Version control — commit .claude/skills to Git so the whole team benefits.
✓
Compose wisely — test for guideline conflicts before combining multiple skills.
